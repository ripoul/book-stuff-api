## What is Book Stuff?

**Book Stuff** is a lightweight reservation platform for shared spaces.
Users create **Places** (a coworking room, a music studio, a holiday home, a workshop...), invite people to access them, and expose any number of bookable **Resources** inside (a desk, a guitar amp, a parking spot...).

The visibility model is built around three actors:

<table>
<tr>
  <td align="center" width="33%"><b>Anonymous visitors</b></td>
  <td align="center" width="33%"><b>Invited members</b></td>
  <td align="center" width="33%"><b>Place managers</b></td>
</tr>
<tr>
  <td align="center">Browse <i>public</i> places & resources, read-only.</td>
  <td align="center">See private places they were invited to, accept / decline invitations.</td>
  <td align="center">Create places, manage resources, send & revoke invitations.</td>
</tr>
</table>

This repo is the **HTTP API** powering it all. The user-facing app lives at
[**ripoul/book-stuff**](https://github.com/ripoul/book-stuff).

## Features

| Domain | What it gives you |
|---|---|
| **Accounts** | Self-service registration, profile updates, JWT authentication (access + refresh). |
| **Places** | CRUD on places, public/private flag, fine-grained per-object permissions via `django-guardian`. |
| **Resources** | CRUD on resources scoped to a place, inheriting place visibility. |
| **Invitations** | Manager invites by email, status state-machine (`pending → accepted / declined / revoked`), automatic permission sync on accept. |
| **Permissions** | Object-level ACLs powered by `django-guardian`, `manage_place` and `can_see` granted automatically. |
| **Filtering** | List endpoints support `django-filter` filters and DRF ordering. |
| **API Schema** | OpenAPI 3 schema and Swagger UI generated by `drf-spectacular`. |
| **CORS** | Pre-configured for `localhost` / `127.0.0.1` to talk to the frontend out of the box. |

## Getting Started

### Requirements

<p>
  <img alt="Python 3.13+" src="https://img.shields.io/badge/Python-%E2%89%A53.13-3776AB?logo=python&logoColor=white">
  <img alt="uv" src="https://img.shields.io/badge/uv-package_manager-DE5FE9?logo=astral&logoColor=white">
  <img alt="Git" src="https://img.shields.io/badge/Git-required-F05032?logo=git&logoColor=white">
</p>

- **Python** `>= 3.13` (managed by [`uv`](https://docs.astral.sh/uv/) via `.python-version`)
- **uv** for dependency management — install it from the [official guide](https://docs.astral.sh/uv/getting-started/installation/)
- **Git** with `pre-commit` hooks installed locally

### Installation

```bash
git clone git@github.com:ripoul/book-stuff-api.git
cd book-stuff-api

uv sync --all-extras

uv run pre-commit install

cd src
uv run python manage.py migrate
uv run python manage.py createsuperuser
uv run python manage.py runserver
```

The API is now live on http://127.0.0.1:8000 and Swagger UI on http://127.0.0.1:8000/docs/swagger/.

### Running the test suite

```bash
cd src
uv run python manage.py test
```

### Tooling

| Tool | Role | Command |
|---|---|---|
| [**uv**](https://docs.astral.sh/uv/) | Python & dependency manager | `uv sync --all-extras` |
| [**ruff**](https://docs.astral.sh/ruff/) | Linter + formatter | `uv run ruff check .` / `uv run ruff format .` |
| [**ty**](https://github.com/astral-sh/ty) | Static type checker | `uv run ty check` |
| [**pre-commit**](https://pre-commit.com/) | Git hook runner | `uv run pre-commit run --all-files` |
| [**Django**](https://www.djangoproject.com/) | Web framework | `uv run python manage.py <command>` |
| [**drf-spectacular**](https://drf-spectacular.readthedocs.io/) | OpenAPI schema | served at `/docs/schema/` |

### Pre-commit hooks

The repo ships with a [`.pre-commit-config.yaml`](./.pre-commit-config.yaml) that runs on every commit:

| Hook | What it does |
|---|---|
| `ruff-check --fix` | Lint Python code and auto-fix what can be fixed |
| `ruff-format` | Format Python code |
| `ty check` | Run static type checking |

Install them once after cloning:

```bash
uv run pre-commit install
```

Run them on demand against the whole repo:

```bash
uv run pre-commit run --all-files
```
